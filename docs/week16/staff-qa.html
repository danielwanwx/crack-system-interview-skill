01

inventory reserve 如何防 oversell？

参考回答
下单时对 SKU/门店执行条件预留，只有 available 数量足够才创建 reservation 并减少可售量。reservation 与 order 关联并有到期/释放路径；浏览库存可缓存，真正结算必须回权威库存。
常见失分
先读库存再异步扣减。
继续追问
多 SKU 订单如何处理部分预留？
02

取消、拣货和库存调整并发谁赢？

参考回答
没有统一“谁赢”，要由状态机定义：未开始拣货可释放 reservation，已拣货可能需要退回或走人工流程。所有命令带 expected version，避免取消覆盖已交付；盘点修正也作为可审计事实进入库存。
常见失分
最后写入覆盖。
继续追问
用户取消时付款已 capture 怎么办？
03

替换如何改变支付？

参考回答
替换先检查用户允许的品类、价格差和上限，生成 substitution proposal；在预授权范围内可自动接受，超出则等待确认。最终 capture 以实际拣货清单计算，所有调整保留 order 和 ledger 证据。
常见失分
直接换商品再扣钱。
继续追问
用户不回复时怎么处理？
04

driver accept timeout 怎么办？

参考回答
调度为司机创建带 expiry 的 offer/reservation，只有在有效版本内 accept 才能进入 assigned。超时或拒绝释放司机并重新找候选；旧 push 的迟到 accept 会被版本检查拒绝。
常见失分
发 push 后无限等待。
继续追问
司机接受但 rider 已取消呢？
05

司机位置是审计数据还是投影？

参考回答
地图上的位置可采样、丢失或晚到，因此适合作为短期读模型和 ETA 输入。取货/送达确认、assignment 和费用等里程碑必须成为 durable 事件；不要用最后一个 GPS 点推断已送达。
常见失分
把所有 GPS 点写订单事务。
继续追问
送达证明如何建模？
06

门店 outage 怎么处理现有订单？

参考回答
先停止新 reservation，已有订单按拣货阶段分类：未拣货可取消释放，已拣货可转店或人工联系。状态变化和退款要可审计，不能让客户只看到静默超时。
常见失分
所有订单统一失败。
继续追问
区域性 outage 如何避免客服雪崩？
07

库存回补的 truth 是什么？

参考回答
库存回补来自收货、盘点、退货和拣货确认等物理事实，而非简单把取消订单加回去。reservation 释放会恢复可售能力，但真实库存偏差需通过 reconciliation 发现和调整。
常见失分
订单表就是库存系统。
继续追问
如何处理损耗？
08

ETA 如何避免伪精确？

参考回答
ETA 应结合门店拣货、司机位置、路况和历史服务时间，并带最后更新时间与不确定性区间。上游状态改变时重算；展示“12:03 准时到”会把预测误差伪装成承诺。
常见失分
只用地图路程。
继续追问
司机未定位时显示什么？
09

为什么判题必须 sandbox？

参考回答
用户程序可以无限循环、fork、读宿主文件或探测内网，所以不能在普通 worker 进程执行。sandbox 将执行隔离到短生命周期环境，并限制资源和系统调用；queue 只能调度，不能阻止恶意代码。
常见失分
说使用 worker pool 就安全。
继续追问
多语言 runtime 如何隔离依赖？
10

sandbox 要限制哪些资源？

参考回答
每个 run 设置 CPU/内存/磁盘/进程数和 wall-clock 限额，限制 syscall 和网络出口，并不挂载其他租户数据或长期 secret。超限时终止并记录原因，销毁环境后再调度下一个任务。
常见失分
只设 timeout。
继续追问
编译阶段与运行阶段的配额为何不同？
11

submission 重试如何不覆盖结果？

参考回答
一次用户提交是稳定 submission，执行重试产生新的 attempt，最终可展示的 result 由受控状态转换决定。worker 失联后重跑不会把较新的完成结果覆盖成旧结果，因为写入带 attempt/version 条件。
常见失分
把结果覆盖在同一行。
继续追问
用户重复点击提交如何处理？
12

judge queue 怎样公平？

参考回答
不同语言编译成本和用户体量差异很大，队列应按语言资源池、租户权重和任务成本调度。单个用户或昂贵语言超额时被限流，但不会阻塞所有轻量题；公平是资源时间而不只是任务数量。
常见失分
全局 FIFO。
继续追问
竞赛高峰如何预留容量？
13

untrusted code 的 secret 攻击面？

参考回答
runner 不应带能访问平台控制面的长期凭证，测试数据和 artifact 也要按任务隔离。通过短期 scoped token、网络 allowlist 和执行后清理，减少代码读取环境变量或横向移动的机会。
常见失分
把所有 secret 放 CI 环境变量。
继续追问
怎样让 judge 下载题目依赖？
14

worker 失联如何清理？

参考回答
scheduler 通过 lease 判断 worker 失联，新的 worker 可重试该 attempt；同时基础设施扫描过期 sandbox、磁盘和临时 token。旧 worker 即使恢复也不能以旧 lease 写最终结果，避免双执行污染。
常见失分
等待 worker 永远回来。
继续追问
执行已产生外部网络副作用怎么办？
15

gateway 与 queue 限流分别保护什么？

参考回答
API gateway 限制提交洪峰、认证滥用和 payload 大小，queue 限制实际 CPU/runner 消耗与租户公平。只做前者会把恶意但已接受的任务堆进后台，只做后者会让控制面先被打垮。
常见失分
一个全局 rate limit 解决所有问题。
继续追问
取消请求应该绕过哪一层？
16

关键 SLO 是什么？

参考回答
配送的核心是库存准确、关键状态推进和 dispatch/取消时延；判题看队列等待、运行超限、sandbox 违规和同输入结果一致性。把它们分别按门店、语言和租户拆分，才能定位局部事故。
常见失分
只看订单 QPS 或 CPU。
继续追问
哪项安全指标应触发暂停某语言？
