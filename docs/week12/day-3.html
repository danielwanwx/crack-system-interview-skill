今日唯一产出物

交付

一张请求/GPU 双状态机时序图，加一份容量预算；包含 admission、优先级队列、batch 形成、prefill/decode、SSE 断连、取消、超时与显存回收。

验收标准

  • 明确 TTFT、inter-token latency、队列等待和吞吐四项指标，能从并发数、平均 token 与 GPU batch 容量反推排队阈值。
  • 普通/付费/重用户公平策略可执行：写权重或配额、饥饿保护、过载拒绝和客户端重试信息。
  • 客户端断连或取消后，控制面能识别 run、传播 cancellation、停止调度并最终释放 GPU；遗漏清理有独立告警。
不通过怎么修

若答案把流式输出等同于 SSE 协议，退回请求状态机，逐步演练断连发生在排队、prefill、decode 三个阶段的状态与资源结果。
