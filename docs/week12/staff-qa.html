01

为什么不只用 SQL 经纬度范围查询？

参考回答
普通范围查询在数据量和热点城市扩大后会扫大量行，难以兼顾文本 filter 和低延迟。geo cell 索引先把空间范围缩成候选集合，再做精确距离计算；代价是要处理 cell 边界和索引维护。
常见失分
给经纬度加普通复合索引就结束。
继续追问
小城市与大城市 cell 粒度是否相同？
02

cell 边界如何避免漏结果？

参考回答
圆形半径可能跨越多个 cell，所以先计算覆盖 cell 集合，尤其包含边界邻居，再用 haversine 或投影距离过滤。这样会多召回一些候选，但不会因为行政边界或 hash 边界漏掉最近商家。
常见失分
只查用户所在 geohash。
继续追问
半径很大时如何控制 cell 数？
03

附近搜索怎样稳定分页？

参考回答
offset 在数据新增和排名变化时会导致重复或漏项，因此 cursor 应编码半径、filter、ranking version 和最后一项的排序键。cursor 有有限有效期；超期后要求刷新，换取可解释性而非承诺永远快照。
常见失分
用 offset 加大 page size。
继续追问
距离相同如何做 tie-break？
04

text 和 geo 谁先执行？

参考回答
如果文本非常具体，可先文本召回再 geo filter；如果地图视窗很小，先 geo 候选更便宜。关键不是死记顺序，而是说明索引选择性、候选上限和 fallback，避免把全量文本和空间扫描相乘。
常见失分
永远先做其中一个。
继续追问
如何处理空文本的“附近餐厅”？
05

rating aggregate 如何维护？

参考回答
新增/隐藏/恢复 review 都产生版本化事件，聚合器按 review_id 去重更新 count、sum 和分布。详情页可在短暂滞后时展示 aggregate 的 as-of 时间；不能每次搜索实时扫描所有评论。
常见失分
只保存平均分。
继续追问
旧评论编辑评分时如何修正？
06

新 review 何时可见？

参考回答
review 通过可见性状态后，详情页可从权威库较快读到，搜索和排序投影可稍后收敛。产品必须区分“已提交”“审核中”“已公开”，避免用户以为所有提交都已影响评分。
常见失分
写入成功即全站同步。
继续追问
商家回复是否也需要审核？
07

删除评论怎样回滚 aggregate？

参考回答
不应物理抹掉事实后再猜 aggregate，而是将 review 标为 hidden/deleted 并发出版本化事件。聚合器减去对应贡献，重复事件按版本去重；这样可以重放并解释为什么评分变化。
常见失分
直接减一并重新平均。
继续追问
涉嫌欺诈但未最终裁定如何展示？
08

商家搬家谁是权威？

参考回答
business 身份通常稳定，地址/location 是带生效时间的可变属性。权威资料更新后，geo index 异步迁移候选 cell；在传播窗口宁可详情正确而搜索短暂旧，且要避免同一商家同时落在两个位置太久。
常见失分
把地址作为 business 主键。
继续追问
连锁店如何建模？
09

热门城市如何避免热点？

参考回答
密集城区的单 cell 会变成热点，应细化层级、按热度拆分或复制读取，并对大视窗限制候选。地图请求还可按 tile 缓存和请求合并；不能只靠全局加机器，因为热点 key 不会自动均匀。
常见失分
固定一个 geohash 精度。
继续追问
节日活动导致瞬时热点怎么办？
10

地图拖动如何减少重复查询？

参考回答
客户端对连续拖动做 debounce，服务端以 tile 或规范化 viewport 缓存结果，并可预取相邻 tile。缓存必须包含 filter 和 ranking 版本；否则看似节省请求却返回错误类别或旧排序。
常见失分
每个像素移动立即查库。
继续追问
用户快速缩放时如何取消旧请求？
11

排名如何可解释？

参考回答
先定义业务排序目标，再以距离、文本匹配、评分、评论量和营业状态组成版本化特征。用户可看到“距离近”或“评分高”等主理由；新商家和付费推广应有明确独立策略，不能悄悄混入自然排名。
常见失分
回答“训练排序模型”。
继续追问
如何避免评论量造成马太效应？
12

怎么做反刷评？

参考回答
先限制异常创建率，记录账号、设备、交易/到店等可解释信号，并让可疑 review 进入 pending 或降低其 aggregate 权重。模型可以辅助排序风险，但最终需要申诉、审计和人工流程；否则误杀无法修正。
常见失分
“用 AI 检测”。
继续追问
商家恶意举报竞争对手怎么办？
13

图片上传增加什么状态？

参考回答
图片要先直传并通过完整性/恶意内容检查，之后才进入 review 的公开版本。对象存储的存在不等于可见性；删除或审核失败需要撤销索引、缩略图和 CDN 副本。
常见失分
把图片 URL 直接写进 review。
继续追问
如何防止重复上传同一违规图片？
14

商户认领如何影响写路径？

参考回答
认领不是给用户一个布尔 owner，而是经过验证的角色授予，带有效期和审计。商户可以修改特定资料或回复评论，但不能改写用户 review 或绕过 moderation；权限事件还要更新缓存和索引。
常见失分
商家登录即可改所有内容。
继续追问
员工离职如何撤权？
15

index schema 如何迁移？

参考回答
先新建索引和 mapping，回填历史并持续双写，再比较 query 质量和延迟后切换读取 alias。旧索引保留回滚期；原地改 schema 往往无法安全处理已有文档。
常见失分
停机重建。
继续追问
review aggregate 字段改语义时如何比较？
16

哪些指标判断系统好用？

参考回答
搜索层看候选覆盖、NDCG、zero-result 与 P99，内容层看审核等待和评分聚合滞后。空间层看 cell 热点、边界补召回率和地图缓存命中；这些共同说明“附近且可信”是否成立。
常见失分
只有 API 成功率。
继续追问
哪项指标会触发回滚新 ranking？
