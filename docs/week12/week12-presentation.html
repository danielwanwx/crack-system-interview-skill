Day 1 · Week 12

Kafka

partition/order； consumer group； ISR 与 durability。
核心 invariant：Kafka 只保证 partition 内顺序；业务顺序键必须稳定映射到同一 partition

参考原文 Kafka Kafka Delivery 本日核心追问
需求边界
  • 先锁定 3 个核心动作：partition/order、consumer group、ISR 与 durability
  • 明确 out of scope，避免把后台运营、推荐、BI 一次性塞进主链路
  • 读写比例、峰值 QPS、对象规模、保留周期必须给数量级假设
  • 先定义用户可见语义，再讨论 Kafka、Redis 或数据库选型
正确性不变量
  • Kafka 只保证 partition 内顺序；业务顺序键必须稳定映射到同一 partition
  • 每个状态转换写 precondition；失败重试不能跳过中间状态
  • 数据库事实、缓存视图、异步派生数据必须标出 authoritative source
  • 重复、乱序、超时、部分成功四类故障逐一给答案
容量与 SLO
  • Week 12 Day 1：先估峰值写 QPS，再按分区键检查热点
  • 在线路径给 p50 / p99；后台 repair 给 freshness 与 backlog SLO
  • 吞吐扩容不能破坏 per-key 顺序、唯一性或余额守恒
  • 压测必须覆盖 skew：最热 1% key 承担 40% 流量
层次 面试中要落到的机制 必须暴露的指标 失败时动作
API 幂等键、分页游标、版本号、错误码 request_p99 / 409_rate 限流、返回可重试语义
Storage 主键、分区键、二级索引、唯一约束 lock_wait / hot_partition 迁移流量或拆热点
Async 事件 ID、顺序键、consumer checkpoint lag / redelivery / DLQ 重放并保持幂等
Derived watermark、版本、last_updated_at freshness / mismatch_rate 从 source of truth 重建
Repair 从 committed offset 重放；幂等 sink 按 event_id/version 去重，坏消息隔离 DLQ stuck_age_p99 / repair_count 先止血，再对账，再回填 
producer.send(topic, key=account_id, value=event,
              enable_idempotence=True, acks='all')

for record in consumer.poll():
    with db.transaction():
        insert_event_if_absent(record.event_id)
        apply_if_newer(record.key, record.version)
        save_offset(record.partition, record.offset)
# commit effect and offset atomically or make effect idempotent

画完组件图就结束；没有唯一约束、状态机、重试边界，也没有说明超时后客户端该不该重试。
Senior 答案：不变量驱动
推荐
围绕「Kafka 只保证 partition 内顺序；业务顺序键必须稳定映射到同一 partition」展开：写入原子点、异步传播、可观测性、自动修复与人工闸门形成闭环。

算法 · Week 12 · Intervals

Intervals
排序、合并与扫描线

区间题先统一闭开语义，再按 start 排序。维护的 invariant 通常是：输出已合并且互不重叠，current 覆盖已扫描前缀的最后一段。

模式 题目 要点
Merge Merge Intervals 按 start 排序维护 current
Insert Insert Interval 左侧/重叠/右侧三段
Sweep Meeting Rooms II 端点事件或 min-heap
Greedy Non-overlapping Intervals 保留结束最早者 
def merge(intervals):
    intervals.sort(key=lambda x: x[0])
    out = []
    for start, end in intervals:
        if not out or start > out[-1][1]:
            out.append([start, end])
        else:
            out[-1][1] = max(out[-1][1], end)
    return out
