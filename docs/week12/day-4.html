今日唯一产出物

交付

一份三组会话 token 账单与上下文策略表；逐组列原始历史、保留/摘要/检索内容、缓存键、失效版本、输入成本、TTFT 与记忆质量检查。

验收标准

  • 至少比较滑动窗口、结构化摘要和按需检索三种策略，给出触发阈值、信息损失与适用会话。
  • 前缀/KV 或响应缓存键包含模型、系统指令、租户/权限及版本；明确何时命中、何时绝不能跨用户复用。
  • 用三组真实长度假设计算 token 与 GPU 时间变化，并设计关键事实召回率或任务完成率作为质量护栏。
不通过怎么修

若成本优化只写“压缩上下文”，选一个包含用户约束和后续指代的长对话；逐句执行策略，证明保留依据并补缓存隔离测试。
