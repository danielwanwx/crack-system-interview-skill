交付

一张包含 agent/remote-write、ingest validation、partitioned TSDB、rollup、query fanout、dashboard cache、rule evaluator 和 notification 的 Metrics Monitoring 白板。

验收标准

  • metric identity、labels、sample timestamp 与 retention 组成清晰数据模型，并有 cardinality guardrail。
  • 写路径和查询路径分离，长时间范围自动选择 rollup 而非扫描原始样本。
  • 告警 evaluation、state、dedupe、silence 和 notification retry 都有持久状态。
不通过怎么修

若设计只有 Kafka、TSDB、Grafana 三个盒子，补写 sample schema、partition key、rollup resolution、rule state 与每段 lag 指标。
