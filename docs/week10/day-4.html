交付

验收标准

  • 响应丢失、producer retry、consumer replay 均不会重复计入业务报表。
  • impression_id 限制同一曝光重复点击，但不把 NAT/IP 当用户身份。
  • 查询低延迟来自预聚合和 partition pruning，且能从 raw events 重算校验。
不通过怎么修

如果声称 exactly once，必须指出 producer ID、log offset/checkpoint、sink key/version 和 raw replay；缺任一项就改称至少一次加幂等效果。

真实追问 2

如何保证不丢点击，又不虚假承诺 exactly once？

触发：候选人说 Kafka exactly once 就结束。

先修：为 producer、log、state、sink、reconcile 五层写唯一 ID 和 checkpoint。

真实追问 3

如何阻止同一用户反复点击广告刷量而不误伤共享 IP？

触发：候选人按 IP 永久去重。

先修：区分 raw click、valid impression、deduped click、billable click 四层。
