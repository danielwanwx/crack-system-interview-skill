Day 1 · Week 10

Geo Indexing

Geohash/S2/H3； 邻格扩展； 精确距离二次过滤。
核心 invariant：粗索引只能召回候选，最终距离必须用原始坐标计算；边界查询不得漏邻格

参考原文 Location Proximity 本日核心追问
需求边界
  • 先锁定 3 个核心动作：Geohash/S2/H3、邻格扩展、精确距离二次过滤
  • 明确 out of scope，避免把后台运营、推荐、BI 一次性塞进主链路
  • 读写比例、峰值 QPS、对象规模、保留周期必须给数量级假设
  • 先定义用户可见语义，再讨论 Kafka、Redis 或数据库选型
正确性不变量
  • 粗索引只能召回候选，最终距离必须用原始坐标计算；边界查询不得漏邻格
  • 每个状态转换写 precondition；失败重试不能跳过中间状态
  • 数据库事实、缓存视图、异步派生数据必须标出 authoritative source
  • 重复、乱序、超时、部分成功四类故障逐一给答案
容量与 SLO
  • Week 10 Day 1：先估峰值写 QPS，再按分区键检查热点
  • 在线路径给 p50 / p99；后台 repair 给 freshness 与 backlog SLO
  • 吞吐扩容不能破坏 per-key 顺序、唯一性或余额守恒
  • 压测必须覆盖 skew：最热 1% key 承担 40% 流量
层次 面试中要落到的机制 必须暴露的指标 失败时动作
API 幂等键、分页游标、版本号、错误码 request_p99 / 409_rate 限流、返回可重试语义
Storage 主键、分区键、二级索引、唯一约束 lock_wait / hot_partition 迁移流量或拆热点
Async 事件 ID、顺序键、consumer checkpoint lag / redelivery / DLQ 重放并保持幂等
Derived watermark、版本、last_updated_at freshness / mismatch_rate 从 source of truth 重建
Repair 抽样全表坐标重算 cell；双写 old/new resolution 并比较召回率 stuck_age_p99 / repair_count 先止血，再对账，再回填 
cell = h3.latlng_to_cell(lat, lng, resolution=9)
neighbors = h3.grid_disk(cell, k=2)
SELECT id, lat, lng FROM places WHERE h3_cell = ANY(:neighbors);
-- exact filter after coarse recall
distance = 2*R*asin(sqrt(haversine_term(a,b)))
return distance <= radius_m

VALUES (:a,:b,'LIKE',NOW())
ON CONFLICT(actor,target) DO UPDATE SET decision='LIKE', ts=NOW();

INSERT INTO matches(user_low,user_high)
SELECT LEAST(:a,:b), GREATEST(:a,:b)
WHERE EXISTS (SELECT 1 FROM swipes WHERE actor=:b AND target=:a AND decision='LIKE')
ON CONFLICT DO NOTHING;

def topo(n, edges):
    graph, indeg = build(edges)
    q = deque(i for i in range(n) if indeg[i] == 0)
    order = []
    while q:
        u = q.popleft(); order.append(u)
        for v in graph[u]:
            indeg[v] -= 1
            if indeg[v] == 0: q.append(v)
    return order if len(order) == n else []
