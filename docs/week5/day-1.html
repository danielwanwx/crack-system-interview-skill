今日唯一产出物

交付

完成一张 Google Docs 端到端设计图：标出 文档快照、操作日志、版本向量与协作者 presence、同步提交边界、主要读写路径、分片键与用户可见状态。

验收标准

  • 按页面顺序从“Understanding the Problem”读到“High-Level Design”，中间标题一个不少。
  • 设计图能逐步解释 并发编辑如何提交、排序、合并并广播给所有协作者 与 加载快照后接续操作日志，实时看到文字、光标与 presence，每条箭头标明协议或数据。
  • 明确 协作服务器为编辑分配 serverVersion 并持久追加到文档日志，并区分事实源、缓存、索引、队列和客户端视图。
  • 至少给出一个数量级估算，并能指出第一个容量瓶颈。
不通过怎么修

若讲解出现组件跳跃，回到 文档快照、操作日志、版本向量与协作者 presence，重画“请求—条件写—派生事件—用户可见结果”的单条时序，再复述一遍。
