今日唯一产出物

交付

完成一张 Rate Limiter 端到端设计图：标出 客户端与规则对应的令牌桶、同步提交边界、主要读写路径、分片键与用户可见状态。

验收标准

  • 按页面顺序从“Understanding the Problem”读到“High-Level Design”，中间标题一个不少。
  • 设计图能逐步解释 一次请求如何被识别、原子扣减并返回 429 或放行 与 在十毫秒预算内读取规则并完成判定，每条箭头标明协议或数据。
  • 明确 Redis 单键原子脚本返回 allow 或 deny，并区分事实源、缓存、索引、队列和客户端视图。
  • 至少给出一个数量级估算，并能指出第一个容量瓶颈。
不通过怎么修

若讲解出现组件跳跃，回到 客户端与规则对应的令牌桶，重画“请求—条件写—派生事件—用户可见结果”的单条时序，再复述一遍。
