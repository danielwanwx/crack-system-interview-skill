2 分钟中文讲译预听

Week 9｜新闻、帖子搜索与爬虫两分钟预听

不自动生成

这一周沿着“发现内容、形成索引、把结果稳定交给用户”这条链学习三个系统。News Aggregator 从数千发布源抓取文章，做规范化、去重、媒体处理和 feed 排序；FB Post Search 把高频帖子与点赞变更投影成倒排索引，同时守住可见性和删除；Web Crawler 则管理十亿级 URL frontier，在故障、重试和 robots.txt 约束下持续推进。三题都不能把 Elasticsearch、Kafka 或 Bloom Filter 当作答案。你必须先指出权威事实在哪里，再画可重放的变更日志、幂等消费者和可重建索引。搜索页可以短暂落后，但隐私变更、删除和 robots 策略要有优先覆盖路径。分页也不是一个 offset 参数：新闻和搜索结果在持续变化，cursor 必须绑定稳定排序键和快照边界，才能减少重复与漏项。深挖时用具体指标检查设计：frontier oldest age、抓取成功率、index lag、权限泄漏、zero-result、热点查询和队列背压。最终目标是能解释一条内容从被发现、被验证、被索引到被展示的完整生命线，并证明任意 worker 崩溃后既不会永久丢失进度，也不会绕过来源方与用户设定的边界。
