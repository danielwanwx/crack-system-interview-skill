2 分钟中文讲译预听

第 5 周导览：两类隔离，两种收敛

不自动生成

本周只有 Google Docs 和 LeetCode，但两题都值得完整两天。Google Docs 训练的是同一文档内的并发编辑：操作怎样获得稳定编号，断线后如何补差，所有客户端怎样收敛，presence 为什么可以丢而正文不可以。LeetCode 训练的是不可信代码隔离：提交怎样进入一次性执行环境，测试如何并行，最终 verdict 由谁唯一决定，十万人比赛怎样在短时间吸收尖峰。通读日必须把接口和高层设计连起来，不要只说协同算法或容器名。Docs 要画快照、操作日志、server version 与 WebSocket 广播；LeetCode 要画 submission、执行尝试、测试结果、verdict 和排行榜事件。深挖日再处理百万连接、日志压缩、旧 owner fencing、资源上限、无外网、执行节点崩溃与榜单收敛。两题的共同方法是把可重试尝试和唯一最终事实分开：编辑可以离线重放但必须按 client sequence 去重，测试可以重复运行但 submission 只能有一个 final verdict。mock 时会给你乱序编辑、超大热文档、恶意死循环和比赛开场洪峰。复盘必须拿指标说话：分叉率、补差大小、隔离违规、判题分歧、最老队列年龄。做完后，你应能解释什么时候追求确定性收敛，什么时候依靠强隔离和异步执行来守住系统边界。使用时先暂停音频，凭记忆画出事实状态、提交点和故障分支，再继续核对。每天结束前必须留下可复查的图、容量计算和一条 repair 记录；如果答案只能说出组件名，就回到原文对应标题，把请求时序与用户可见结果补全。周末的综合 mock 不追求背诵固定架构，而要在约束变化后仍能守住不变量、说明取舍，并用指标决定降级、扩展和回滚。
