Senior / Staff pressure test

先答，再看完整 Q&A

真实追问 1

两位用户同时右滑，如何保证只产生一个 Match？

触发：候选人先读反向 swipe 再直接 insert，没有并发保护。

先修：并排画两个 swipe 事务，在 Match insert 处标出唯一键、winner 与 loser read。

真实追问 2

候选栈如何在低延迟与最新位置之间取舍？

触发：候选人只选预计算或只选在线查询，没有陈旧边界。

先修：把候选生成拆成离线 pool、在线 filter、refill 三步并标出版本。

真实追问 3

数亿次历史 swipe 怎样避免重复展示，又不让查询变慢？

触发：候选人把完整 ID 集合塞进单个请求或单个 Redis key。

先修：为近期、长期、block 三类集合分别写存储、精度和回源策略。
