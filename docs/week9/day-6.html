今日唯一产出物

交付

lease/checkpoint 恢复、host politeness scheduler、URL/content dedupe、10B/5d 容量表、毒页面与 recrawl runbook。

验收标准

  • worker 在 fetch、store 或 parse 任一点崩溃都不会永久丢 URL，重复执行安全。
  • robots 与 host rate limit 不依赖 worker 自觉，跨集群仍有单一调度约束。
  • 吞吐、带宽、存储、frontier size 和失败重试能证明五天目标是否可行。
不通过怎么修

答不出规模时用 10B/(5×86400) 算最低 page/s，再乘平均字节和失败率；答不出恢复时逐步模拟 lease 过期和重复 fetch。

真实追问 1

worker 下载页面后崩溃，如何既不丢进度又不重复污染数据？

触发：候选人说消息 ack，却没有 ack 前后的故障窗口。

先修：列出 fetch、body put、metadata、discover、ack 五个崩溃点和重试结果。

真实追问 2

数千 worker 如何共同遵守同一网站的 robots.txt 和 crawl-delay？

触发：候选人在每个 worker 内 sleep。

先修：画 host bucket、permit、worker、feedback 四步，模拟两个 worker 竞争。

真实追问 3

10B 页面五天抓完，需要多少吞吐，哪个资源最可能先触顶？

触发：候选人直接说横向扩展，没有数量级。

先修：完成 page/s、bytes/s、concurrency、frontier writes 四行容量表。
