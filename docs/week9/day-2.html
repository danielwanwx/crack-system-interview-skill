今日唯一产出物

交付

稳定 cursor、feed cache/materialization、30 分钟 ingestion SLO、媒体 CDN、breaking-news 热点与 personalization fallback 矩阵。

验收标准

  • cursor 能在新文章持续插入时解释重复、漏项和 snapshot 边界。
  • 从 source polling 到 feed 可见的每段 lag 均可测量并能定位瓶颈。
  • 热点、个性化服务故障和媒体处理积压都有明确降级而不阻断基础 feed。
不通过怎么修

如果只会说 cache feed，拆成 ingest age、dedupe lag、index lag、ranking latency、cache age 五段，并为每段写 owner、SLO 和 fallback。

Senior / Staff pressure test

先答，再看完整 Q&A

真实追问 1

新闻不断插入时，cursor 如何减少重复和漏项？

触发：候选人仍使用 page/offset，或 cursor 只保存 article_id。

先修：用三篇新文章插入前后演算 offset 与复合 cursor 的结果差异。

真实追问 2

如何把 feed p95 控制在 200ms，而不是只说加缓存？

触发：候选人没有候选准备、预算拆分或 fallback。

先修：画 200ms waterfall，并给每个远程调用 timeout 与 fallback。

真实追问 3

如何证明文章在发布后 30 分钟内进入 feed？

触发：候选人只提高 polling 频率，没有端到端时间线。

先修：为一篇文章填完整阶段时间戳并设计三条阶段报警。
