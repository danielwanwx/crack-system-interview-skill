Day 1 · Week 9

搜索索引基础

倒排索引与 postings； analyzer/tokenizer； BM25 排序。
核心 invariant：索引文档版本不得回退；query 结果可陈旧但不能混入已删除且受权限保护的文档

参考原文 Search Elasticsearch 本日核心追问
需求边界
  • 先锁定 3 个核心动作：倒排索引与 postings、analyzer/tokenizer、BM25 排序
  • 明确 out of scope，避免把后台运营、推荐、BI 一次性塞进主链路
  • 读写比例、峰值 QPS、对象规模、保留周期必须给数量级假设
  • 先定义用户可见语义，再讨论 Kafka、Redis 或数据库选型
正确性不变量
  • 索引文档版本不得回退；query 结果可陈旧但不能混入已删除且受权限保护的文档
  • 每个状态转换写 precondition；失败重试不能跳过中间状态
  • 数据库事实、缓存视图、异步派生数据必须标出 authoritative source
  • 重复、乱序、超时、部分成功四类故障逐一给答案
容量与 SLO
  • Week 9 Day 1：先估峰值写 QPS，再按分区键检查热点
  • 在线路径给 p50 / p99；后台 repair 给 freshness 与 backlog SLO
  • 吞吐扩容不能破坏 per-key 顺序、唯一性或余额守恒
  • 压测必须覆盖 skew：最热 1% key 承担 40% 流量
层次 面试中要落到的机制 必须暴露的指标 失败时动作
API 幂等键、分页游标、版本号、错误码 request_p99 / 409_rate 限流、返回可重试语义
Storage 主键、分区键、二级索引、唯一约束 lock_wait / hot_partition 迁移流量或拆热点
Async 事件 ID、顺序键、consumer checkpoint lag / redelivery / DLQ 重放并保持幂等
Derived watermark、版本、last_updated_at freshness / mismatch_rate 从 source of truth 重建
Repair CDC offset + document version 检测漏更，按 DB snapshot 重建新索引后 alias swap stuck_age_p99 / repair_count 先止血，再对账，再回填

{ title, body, tenant_id, visibility, updated_at }

GET /documents/_search
{ "query":{"bool":{"must":{"match":{"body":"vector database"}},
                    "filter":{"term":{"tenant_id":"t7"}}}} }
脆弱答案：只讲 happy path
画完组件图就结束；没有唯一约束、状态机、重试边界，也没有说明超时后客户端该不该重试。
Senior 答案：不变量驱动
推荐
围绕「索引文档版本不得回退；query 结果可陈旧但不能混入已删除且受权限保护的文档」展开：写入原子点、异步传播、可观测性、自动修复与人工闸门形成闭环。

算法 · Week 9 · Backtracking

Backtracking
选择、约束、撤销

把搜索空间写成决策树：state 表示已做选择，choices 产生分支，constraint 尽早剪枝，返回时恢复现场。

模式 题目 要点
子集 Subsets 每层决定选/不选
排列 Permutations used 集合防重复使用
组合 Combination Sum start index + remaining 剪枝
棋盘 N-Queens 列与两条对角线集合 
def dfs(start, path):
    # invariant: path is a valid partial solution
    if goal(path):
        ans.append(path.copy()); return
    for i in range(start, len(candidates)):
        if violates(path, candidates[i]): continue
        path.append(candidates[i])       # choose
        dfs(next_start(i), path)
        path.pop()                       # unchoose

口述模板
  • 先定义状态与返回值，不边写边猜
  • 给出循环/递归 invariant，再说明终止条件
  • 主动覆盖空输入、重复值、溢出与退化结构
  • 最后报时间、空间复杂度及可替代方案
Mock 标准
  • 3 分钟澄清 + 5 分钟方案 + 20 分钟代码
  • 每 5 分钟口头同步进度，不沉默写码
  • 手跑最小、边界、对抗三个 case
  • 修 bug 时先定位 invariant 被哪一行破坏
去重 排序 + 同层跳过相同值 避免重复解
剪枝 remaining<0 / 冲突集合 越早越省指数分支
复杂度 O(branch^depth) 同时说明输出规模下界
