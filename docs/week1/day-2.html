真实追问 1

为什么不直接对 long URL 做 hash 截断成 short code？

触发：候选人说“hash 一下就唯一”，没有讨论碰撞或相同 URL 的产品语义。

先修：画出 generate → conditional insert → conflict/retry → response 丢失后的完整状态机。

真实追问 2

redirect path 为什么选择 302，而不是 301？

触发：候选人只说两者都是重定向，或认为 301 一定更快。

先修：列出 301/302 对控制权、延迟、统计和失效窗口的四列表。

真实追问 3

一个明星短链突然成为热点，缓存已经有值，为什么还可能出问题？

触发：候选人认为 cache hit 就意味着热点已经解决。

先修：画出热点 key 在 edge、process cache、Redis 和 DB 四层的命中与失效时序。
