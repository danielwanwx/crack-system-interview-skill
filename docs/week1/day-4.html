真实追问 1

为什么不让文件先经过应用服务器再写对象存储？

触发：候选人画了 client → API server → S3 的整文件数据路径。

先修：画 control plane 与 data plane，并标出每一步授权、hash 和状态。

真实追问 2

对象已上传成功，但 metadata commit 超时，系统现在是什么状态？

触发：候选人说用分布式事务，或直接重新上传。

先修：写出 uploading、uploaded、committed、expired 四状态和幂等转移。

真实追问 3

如果 WebSocket 通知丢了，为什么你仍能保证设备最终同步？

触发：候选人把 WebSocket/Kafka 说成唯一同步来源。

先修：画通知丢失后的 pull replay 时序，并定义 full resync 触发条件。
