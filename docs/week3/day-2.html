Senior / Staff pressure test

先答，再看完整 Q&A

真实追问 1

Redis Pub/Sub 在发布瞬间丢了一条消息，为什么在线用户最终仍能收到？

触发：候选人把 Redis 频道当消息事实库，或无法解释订阅者断开时的数据去向。

先修：把发送时序改成 Inbox commit→publish，并演练 publish 丢失、重复和聊天沉默三条恢复路径。

真实追问 2

同一用户有手机、平板和电脑，其中电脑离线一周；你的投递状态怎样建模？

触发：候选人仍使用 userId 级单一已送达标记，无法说明某一设备补消息。

先修：为三个 deviceId 各画 Inbox 水位，再用注销和新增设备验证 membershipVersion。

真实追问 3

一台 Chat Server 维持一百万连接时突然宕机，怎样避免重连把剩余集群再次打垮？

触发：候选人只说负载均衡会把客户端分到其他服务器，没有重连控制与容量余量。

先修：做一次百万连接断开的故障演算，分别限制握手、鉴权、注册和历史回放速率。
