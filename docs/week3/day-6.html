Senior / Staff pressure test

先答，再看完整 Q&A

真实追问 1

白方手机和网页几乎同时提交两步不同的棋，你怎样保证最多接受一手？

触发：候选人把并发落子错误地引用到 matchmaking，或只说 WebSocket 有序。

先修：用两个 expectedMoveNumber=26 的请求画条件提交，明确第二个返回 conflict 与最新状态。

真实追问 2

两个玩家 RTT 分别为二十毫秒和三百毫秒，服务器怎样扣钟才不让客户端作弊？

触发：候选人信任客户端上报剩余时间，或完全忽略不对称网络造成的体验差异。

先修：用一回合时间线计算无补偿、有限补偿和恶意客户端三种结果。

真实追问 3

承载一万盘棋的 Game Server 宕机后，如何恢复局面、时钟和连接而不产生两个 owner？

触发：候选人只说客户端重连到另一台机器，没有持久状态与所有权 epoch。

先修：故障演练必须包含 lease 过期、新 epoch、快照重放、客户端补步和旧 owner 晚写。
